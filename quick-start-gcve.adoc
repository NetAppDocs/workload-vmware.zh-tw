= 使用 VMware 工作負載遷移顧問快速入門遷移至 Google Cloud VMware Engine
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


[role="lead"]
使用 VMware 遷移顧問，將您目前的基礎架構遷移到 Google Cloud VMware Engine，並使用 Google Cloud NetApp Volumes 作為外部 NFS 資料儲存。

在開始之前、您應該先了解 https://docs.netapp.com/us-en/workload-setup-admin/permissions-reference.html["NetApp Workload Factory 的權限"^]。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-1.png["一"]登入工作負載工廠
[role="quick-margin-para"]
你需要 https://docs.netapp.com/us-en/workload-setup-admin/sign-up-saas.html["在 Workload Factory 上建立帳戶"^]並使用以下任一方式登入 https://docs.netapp.com/us-en/workload-setup-admin/console-experiences.html["主控台體驗"^]。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-2.png["二"]新增憑證和權限
[role="quick-margin-para"]
選擇 https://docs.netapp.com/us-en/workload-setup-admin/permissions-reference.html["權限策略"]滿足您的需求。

[role="quick-margin-para"]
如果您選擇不授予權限，則可以開始使用 Workload Factory for VMware 來複製部分完成的程式碼範例。

[role="quick-margin-para"]
如果您選擇授予權限，則需要 https://docs.netapp.com/us-en/workload-setup-admin/add-credentials.html["手動向帳戶新增憑證"]提供具有檢視和管理 Google Cloud NetApp Volumes 資源、VPC 配置以及 Google Cloud VMware Engine 資料儲存區連線所需權限的 GCP 服務帳戶。

[role="quick-margin-para"]
https://docs.netapp.com/us-en/workload-setup-admin/add-credentials.html["瞭解如何新增認證和權限"^]。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-3.png["三"]上傳虛擬機器清單數據
[role="quick-margin-para"]
您可以使用規劃中心上傳有關目前 VM 環境的庫存詳細資訊。使用資料收集器腳本、RVTools 或NetApp Data Infrastructure Insights擷取目前的 VM 配置並將資料上傳到規劃中心。

[role="quick-margin-para"]
link:upload-vm-inventory.html["了解如何上傳虛擬機器清單數據"^] 。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-4.png["四"] 使用 VMware 移轉顧問建立部署計畫
[role="quick-margin-para"]
啟動 VMware migration advisor，並使用 Google Cloud NetApp Volumes 作為外部 NFS 資料存放區，配置您想要在 Google Cloud VMware Engine 基礎架構上建立的 VM 環境。

[role="quick-margin-para"]
link:launch-migration-advisor-gcve.html["瞭解如何使用 VMware 移轉顧問來建立部署計畫"].
