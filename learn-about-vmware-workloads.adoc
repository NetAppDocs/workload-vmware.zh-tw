= 了解NetApp Workload Factory for VMware
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


[role="lead"]
NetApp Workload Factory for VMware 提供工具，可將您的資料從本機系統移轉到 Amazon Elastic VMware Service （EVS）、VMware Cloud on AWS （VMC）、Amazon EC2 或 Google Cloud VMware Engine。



== 什麼是NetApp Workload Factory for VMware？

NetApp Workload Factory for VMware 提供規劃中心和遷移顧問，協助您分析本地 vSphere 環境中的目前虛擬機器配置。遷移顧問隨後會產生一個計劃，將推薦的虛擬機器佈局部署到 Amazon EVS、Amazon EC2、VMware Cloud on AWS vSphere 叢集或 Google Cloud VMware Engine，並使用自訂的雲端原生儲存作為外部資料儲存。對於 AWS 環境，這包括 Amazon FSx for NetApp ONTAP 檔案系統。對於 Google Cloud 環境，這包括 Google Cloud NetApp Volumes。規劃中心作為 Workload Factory for VMware 的控制面板，可讓您管理已儲存的虛擬機器清單資料集和已儲存的遷移計劃，並配置任何已儲存的遷移計劃。

Amazon FSx for NetApp ONTAP 是一個建構於 NetApp 的 ONTAP 檔案系統上的外部 NFS 資料存放區，可以掛載到 Amazon EVS 或 Amazon EC2 執行個體，或 VMware Cloud on AWS vSphere 叢集。同樣地，Google Cloud NetApp Volumes 為 Google Cloud VMware Engine 環境提供外部 NFS 資料存放區。無需新增更多主機來增加可用儲存空間，只需使用這些雲端原生儲存磁碟區作為外部資料存放區，以補充 vSAN 資料存放區。這為您提供一個靈活、高效能且虛擬化的儲存基礎架構，能夠獨立於運算資源進行擴展。

有關 Workload Factory 的更多信息，請參閱 https://docs.netapp.com/us-en/workload-setup-admin/workload-factory-overview.html["工作負載工廠概述"^]。



== VMware 遷移顧問的工作原理

遷移顧問可以幫助您將運行在任何 VMware 支援的資料儲存上的本機虛擬機（VM）及其資料遷移到 Amazon EVS、Amazon EC2、VMware Cloud 或 Google Cloud VMware Engine 資料存儲，其中包括在 FSx for ONTAP 檔案系統或 Google Cloud NetApp Volumes 上的補充 NFS 資料儲存。

請注意、您最多可以將四（ 4 ）個適用於 ONTAP 磁碟區的 FSX 連接至 VMware Cloud 上 AWS 上的單一 vSphere 叢集。



== 移民顧問能為您做些什麼

遷移顧問提供以下功能：

* 分析目前的內部部署 VM 組態
* 決定要將哪些虛擬機器移轉到 Amazon EC2、Amazon EVS、VMware Cloud on AWS 或 Google Cloud VMware Engine
* 確定用作 VM 外部資料存放區所需的雲端原生儲存磁碟區空間
* 檢閱結果報告以瞭解部署步驟
* 執行實際部署（適用於 Amazon EC2、Amazon EBS 和 VMware Cloud on AWS）


遷移顧問支援針對單一 Amazon EVS、Amazon EC2 或 VMware Cloud on AWS 叢集，連接至 Amazon FSx for NetApp ONTAP 檔案系統，或連接至 Google Cloud VMware Engine 叢集與 Google Cloud NetApp Volumes 的組態規劃。



== 使用移民顧問的好處

將您目前基礎架構的部分內容移轉到 Amazon EC2、Amazon EBS、VMware Cloud on AWS 或 Google Cloud VMware Engine，並使用雲端原生儲存作為外部 NFS 資料存放區，可帶來以下好處：

* 由於主機和儲存設備分離、以及進階資料效率、因此能達到成本最佳化
* 可視需要擴充儲存容量、無需購買額外的主機執行個體
* 雲端中的 NetApp ONTAP 資料管理功能、例如節省空間的快照、複製、壓縮、重複資料刪除、壓縮、 和複寫
* 減少硬體更新的管理
* 除了增加或減少磁碟區大小之外、還能變更資料處理量、 IOPS 和檔案系統大小
* 高可用度支援多個可用性區域（ AZ ）部署
* 使用 VPC 對等的單一 AZ 組態、無需使用 Transit Gateway 、即可降低成本和延遲




== 使用NetApp Workload Factory 的工具

您可以將NetApp Workload Factory 與以下工具一起使用：

* *Workload Factory 控制台*：Workload Factory 控制台提供您的應用程式和專案的視覺化、整體視圖。
* * NetApp控制台*： NetApp控制台提供混合介面體驗，以便您可以將 Workload Factory 與其他NetApp資料服務一起使用。
* *問我*：使用問我 AI 助理來提問並了解有關 Workload Factory 的更多信息，而無需離開 Workload Factory 控制台。從 Workload Factory 幫助選單中存取「問我」。
* *CloudShell CLI*：Workload Factory 包含 CloudShell CLI，可透過基於瀏覽器的單一 CLI 跨帳號管理和操作 AWS 和NetApp環境。從 Workload Factory 控制台頂部欄存取 CloudShell。
* *REST API*：使用 Workload Factory REST API 部署和管理您的 FSx for ONTAP檔案系統和其他 AWS 資源。
* *CloudFormation*：使用 AWS CloudFormation 程式碼執行您在 Workload Factory 控制台中定義的操作，以從您的 AWS 帳戶中的 CloudFormation 堆疊對 AWS 和第三方資源進行建模、配置和管理。
* *Terraform NetApp Workload Factory 提供者*：使用 Terraform 建置和管理在 Workload Factory 控制台中產生的基礎架構工作流程。




== 針對 EVS 環境的架構完善分析

「架構完善」標籤可對您的 Amazon Elastic VMware Service (EVS) 環境進行每日自動分析，以確保其符合 AWS 和 NetApp 最佳實務。此標籤可識別配置問題，並提供詳細的修復指南，幫助您維持最佳的運作、安全性和成本效益。

自動掃描使用 AWS API 執行，無需 vSphere 憑證。掃描結果按配置區域進行組織，每項結果都包括狀態、嚴重程度、受影響資源的詳細資訊以及逐步修復程序。

== 區域

所有支援 FSx for ONTAP 的商業區域均支援 Workload Factory。 https://aws.amazon.com/about-aws/global-infrastructure/regional-product-services/["看看支援的亞馬遜區域。"^]

以下 AWS 區域不受支援：

* 中國地區
* GovCloud（美國）區域
* 秘密雲
* 絕密雲
